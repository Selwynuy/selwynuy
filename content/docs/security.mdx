## The principle

Security is usually bolted on at the end, under deadline pressure, after an audit hands you a list. I start from the opposite end. The defaults are chosen so that the insecure version is the one you have to go out of your way to write.

## What "secure by default" means in practice

- **Secrets never reach the client.** API keys and tokens are read on the server only. A `NEXT_PUBLIC_` prefix is a decision to expose a value to the browser, made on purpose, never by accident.
- **Validate every input at the boundary.** Data from a request is untrusted until it has been parsed and validated. The server is where that happens, before anything touches a database.
- **Least privilege everywhere.** Every credential, every database role, every access policy grants the minimum needed and nothing more.
- **Authentication and authorization are checked on the server.** A hidden button is not access control. The check that matters runs where the user cannot tamper with it.
