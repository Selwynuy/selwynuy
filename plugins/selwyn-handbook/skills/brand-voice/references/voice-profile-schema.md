# Voice Profile Schema

Use this exact structure when building a reusable voice profile from source material:

```text
VOICE PROFILE
=============
Source:
Confidence:

Source Set
- source 1
- source 2
- source 3

Rhythm
- short note on sentence length, pacing, and fragmentation

Compression
- how dense or explanatory the writing is

Capitalization
- conventional, mixed, or situational

Parentheticals
- how they are used and how they are not used

Question Use
- rare, frequent, rhetorical, direct, or mostly absent

Claim Style
- how claims are framed, supported, and sharpened

Preferred Moves
- concrete moves the source does use

Banned Moves
- specific patterns the source does not use

Channel Notes
- Article/docs:
- Social:
- Email:
```

## Guidelines

- Keep the profile concrete and source-backed. Every line should trace to something in the source set, not a general impression.
- Use short bullets, not essay paragraphs. The point is operational reuse in a drafting session, not literary criticism.
- Every banned move should be observable in the source set or explicitly requested. Don't invent bans that aren't backed by evidence.
- If the source set conflicts (public launch voice vs. private working voice, old vs. new), call out the split instead of averaging it into mush.
- If no source material was available and the default voice from `SKILL.md` was used instead, write `Source: default (no material provided)` so that's visible to whoever reads the profile later.
